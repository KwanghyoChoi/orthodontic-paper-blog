## 실행 방법

### Phase 1: 콘텐츠 생성
```bash
claude "논문 분석 시작: input/[논문파일명].pdf"
```

### Phase 2: 발행 (Human 승인 후)
```bash
python tools/publish_blog.py output/[블로그파일].md --publish
```

## 에이전트 역할

| 에이전트 | 역할 | 모델/도구 |
|---------|------|----------|
| paper_analyzer | 논문 핵심 추출 + 복잡도 판단 | Claude |
| research_expander | 관련/반박 연구 검색 | Sonar Deep Research |
| comparator | 연구 간 비교 + 논쟁점 분석 | Claude |
| image_curator | Figure 식별 + 크롭 + **검증 루프** | Claude Vision |
| blog_writer | 블로그 초안 + **한걸음 더 깊이 들여다보기** | Sonnet 4.5 |
| quality_reviewer | 품질 검토 + 재호출 판단 | Claude |
| blog_publisher | WordPress 발행 | REST API |

## ⚠️ 필수 워크플로우 (2025 업데이트)

### 1. 블로그 구조 순서

```
원논문 내용 (연구 배경 → 방법 → 주요 발견 → 실제 진료)
    ↓
"정리" 섹션 (핵심 요약)
    ↓
"🔬 한걸음 더 깊이 들여다보기" 섹션 (심화 분석)
    ↓
"참고문헌" 섹션
```

### 2. "한걸음 더 깊이 들여다보기" 섹션 (blog_writer)

블로그 **"정리" 섹션 다음, "참고문헌" 앞에** 반드시 포함:

```markdown
## 🔬 한걸음 더 깊이 들여다보기

> 💡 이 섹션은 논문 내용을 넘어선 심화 분석입니다

### 최신 연구들은 어떻게 보나?
- 구체적 수치와 함께 비교 분석
- 비교 테이블 필수

| 항목 | 본 논문 | 다른 연구 | 해석 |
|-----|--------|----------|-----|
| ... | ... | ... | ... |

### 왜 그럴까? - 메커니즘 이해하기
- 핵심 현상의 생역학적/생물학적 원리
- 연구 결과 인용 포함

### 그래서 어떻게? - 임상 전략
- 구체적 해결책과 프로토콜
- 실전 적용 가이드

### 전문가들의 시선
- 합의문, 가이드라인 인용
- 논쟁점과 현재 견해

### 앞으로의 방향
- 최신 연구 동향
- 새로운 기술/접근법
```

**❌ 잘못된 예:** 참고문헌에만 논문 나열하고 본문에서 언급 안 함
**✅ 올바른 예:** 본문에서 구체적 수치, 메커니즘, 임상 전략까지 심화 분석

### 3. 이미지 크롭 검증 루프 (image_curator)

**⚠️ 필수 크롭 대상:**
- [ ] **논문 커버 (Page 1)** - 대표이미지(featured_image) + "원논문 소개" 섹션
- [ ] **핵심 Figure들** - PRISMA, Forest plot, 차트, 다이어그램 등

**검증 체크리스트:**
- [ ] Figure 내용이 모두 포함되었는가? (a,b,c... 모든 패널)
- [ ] 캡션이 잘리지 않았는가?
- [ ] 다이어그램의 축, 레이블이 완전한가?
- [ ] 불필요한 여백이 과도하지 않은가?
- [ ] 논문 커버가 제목/저널명/저자를 포함하는가?

**좌표 조정 가이드:**
| 문제 | 해결 |
|-----|-----|
| 하단 잘림 | bottom 값 증가 |
| 상단 잘림 | top 값 감소 |
| 캡션 누락 | bottom을 캡션 끝까지 확장 |

## 주요 기능

- **이미지 최적화**: PNG → WebP (약 59% 용량 감소)
- **이미지 호스팅**: Google Drive (lh3.googleusercontent.com)
- **대표이미지**: FIFU 플러그인 (논문 커버 Page 1 사용)
- **SEO**: Rank Math Focus 키워드 자동 설정
- **카테고리**: "최신 치과교정학 연구" 자동 분류

## 발행 전 체크리스트

### 콘텐츠 구조
- [ ] 서두에 저자 자기소개 (가볍게)
- [ ] 마지막에 저자 정보 (이름, 전문의, 병원, 지역)
- [ ] 원논문 정보 정확히 명시 (저자, 제목, 저널, DOI)
- [ ] 3,000자 이상
- [ ] "정리" 섹션 포함
- [ ] "🔬 한걸음 더 깊이 들여다보기" 섹션 포함 (정리 다음, 참고문헌 앞)

### 이미지 ⚠️
- [ ] 논문 커버 (Page 1) 크롭됨
- [ ] 논문 커버가 대표이미지(featured_image)로 설정됨
- [ ] 논문 커버가 "원논문 소개" 섹션에 삽입됨
- [ ] 모든 Figure가 Vision으로 검증됨 (잘림 없음)
- [ ] 모든 이미지가 WebP로 변환됨
- [ ] 모든 이미지가 Google Drive에 업로드됨
- [ ] 모든 이미지 URL이 블로그에 삽입됨
- [ ] 모든 이미지에 출처(citation) 표기됨

### 발행 후
- [ ] `rm -rf tmpclaude-*` 실행 (임시 파일 정리)
